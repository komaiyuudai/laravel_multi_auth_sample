Laravel Multi Authについて

Multi Auth

・Laravel5.2より基本仕様となった
・複数種類のアカウントの管理が可能（例：user、admin）
・config/auth.phpに設定を記述

必須カラム

  • password varchar
    パスワード保存用カラム、最低でも60文字は入るように
  • remember_token varchar
    セッションのトークンを保存するカラム、100文字は入るように
  • 認証用のカラム
    デフォルトではemail、変更可

前提
  • Controller、Viewを新たに生成する
  • 基本的に php artisan make:auth で作られたファイルをコピーして使用

auth.php

authの設定を記述するファイル。
使用する認証アカウントや、それに対応するモデルの定義を行う

  • defaults：デフォルトで指定されるガード＆パスワード設定
  • guards：アカウントの種類のようなもの
  • providers：対応するモデルの設定（guardsで指定する）
  • passwords：パスワードリセット機能を使用したい場合の設定
# デフォルト設定
'defaults' => [
    'guard' => 'web',
    'passwords' => 'users',
],

# ガード設定
'guards' => [
    'web' => [
        'driver' => 'session',
        'provider' => 'users',
    ],
    'api' => [
        'driver' => 'token',
        'provider' => 'users',
    ],
    'admin' => [
        'driver' => 'session',
        'provider' => 'admin',
    ],
],

# プロバイダー（モデル）設定
'providers' => [
    'users' => [
        'driver' => 'eloquent',
        'model' => App\User::class,
    ],
    'admin_accounts' => [
        'driver' => 'eloquent',
        'model' => App\Admin::class,
    ],
],

# パスワード設定
'passwords' => [
    'users' => [
        'provider' => 'users',
        'email' => 'user.emails.password',
        'table' => 'password_resets',
        'expire' => 60,
    ],
    'admin' => [
        'provider' => 'admin',
        'email' => 'admin.emails.password',
        'table' => 'password_resets',
        'expire' => 60,
    ],
],

/**
 * view変更の為
 */
public function showLinkRequestForm()
{
    if (property_exists($this, 'linkRequestView')) {
        return view($this->linkRequestView, [
            'guard' => $this->broker,
        ]);
    }

    if (view()->exists('admin_accounts.passwords.email')) {
        return view('admin_accounts.passwords.email', [
            'guard' => $this->broker,
        ]);
    }

    return view('admin_accounts.password', [
        'guard' => $this->guard,
    ]);
}

/**
 * view変更の為
 */
public function showResetForm(Request $request, $token = null)
{
    // viewでguard判別ができるように
    $guard = $this->broker;

    if (is_null($token)) {
        return $this->getEmail();
    }

    $email = $request->input('email');

    if (property_exists($this, 'resetView')) {
        return view($this->resetView)->with(compact('token', 'email'));
    }

    if (view()->exists('admin_accounts.passwords.reset')) {
        return view('admin_accounts.passwords.reset')->with(compact('token', 'email', 'guard'));
    }

    return view('admin_accounts.reset')->with(compact('token', 'email', 'guard'));
}

懸念点

Multi Auth使用時、auth.php内でdefault指定している種類のアカウントがログイン状態になると、
別種類のアカウントでもログイン状態となってしまい、ログイン画面にアクセスしようとしてもTOPにリダイレクトされてしまう。
app/Http/Middleware/RedirectIfAuthenticated.php（guestミドルウェア）が使われている。

例）

default = user
user = ログイン状態
admin = ログインできない（リダイレクトされる）

default = user
user = ログインできる
admin = ログイン状態
auth.php
'defaults' => [
    'guard' => 'user',
    'passwords' => 'user',
],
'guards' => [
    'user' => [
        'driver' => 'session',
        'provider' => 'users',
    ],

    'admin' => [
        'driver' => 'session',
        'provider' => 'admin',
    ],
]
routes.php
Route::get('login', 'Admin/AuthController@showLoginForm');
Route::post('login', 'AdminAccount\AuthController@login');

上記を使用すると、上に書いた現象が起こる。
上記はlaravelのログイン機能をそのまま使用しているのだが、guestミドルウェアを指定しなくてもリダイレクトされてしまうので、
そういう仕様になっている？

解決法
  1. ログインフォーム/ログイン処理の自作（デフォルトのやつを使わない）
  2. defaultにダミーアカウントを設定する（テーブル等も必要）
  3. guestミドルウェアをカスタム（複数種類アカウントのログインを不可能に）
  4. guestミドルウェアをカスタム（複数種類アカウントのログインを可能に）
試してみて
  1. 新しくコントローラを作成し、その中にログインフォーム/ログイン処理を記述した。動作的には問題なし
  2. ダミーアカウント登録だと、ふとした拍子にエラーになる可能性あり（しっかり設定しておけばok？）。使用しない設定やテーブルがあるので初見だと混乱するかも。

  3. guardを定数に定義しておき、ミドルウェア内でどれかログインしていたらリダイレクトさせた。動作的には問題なし。

    RedirectIfAuthenticated.php

    • before

      public function handle($request, Closure $next, $guard = null)
{
   if (Auth::guard($guard)->check()) {
       return redirect('/');
   }
   return $next($request);
}

    • after

      public function handle($request, Closure $next)
{
   foreach (Config::get('const.GUARD') as $guard) {
       if (Auth::guard($guard)->check()) {
           return redirect('/');
       }
   }
   return $next($request);
}

  4. guardがnullの時はリダイレクトしないようにする。これが一番良いかも

    RedirectIfAuthenticated.php

    • before

      public function handle($request, Closure $next, $guard = null)
{
   if (Auth::guard($guard)->check()) {
       return redirect('/');
   }
   return $next($request);
}

    • after

      public function handle($request, Closure $next, $guard = null)
{
   if ($guard != null && Auth::guard($guard)->check()) {
       return redirect('/');
   }
   return $next($request);
}
